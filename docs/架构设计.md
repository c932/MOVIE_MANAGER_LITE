# 架构设计

## 1. 模块划分

- `ui`: 界面层（主窗口、卡片、详情、设置）
- `models`: 数据模型
- `parsers`: NFO 解析
- `scraper`: 刮削工作流与对话框
- `utils`: 配置、缓存、收藏、观看历史、路径工具
- `tools/diagnostics`: 诊断脚本

## 2. 核心流程

1. `main.py` 启动应用
2. `ConfigManager` 读取配置
3. `LibraryScanner` 扫描媒体并解析 NFO
4. UI 渲染海报墙与筛选器
5. 用户行为触发收藏、观看状态与播放

## 3. 数据存储策略

统一存放到 `data` 目录（由 `utils/app_paths.py` 管理）：

- `data/config.json`
- `data/movie_cache.json`
- `data/watch_history.json`
- `data/favorites.json`
- `data/failed_movies.json`
- `data/scrape_config.json`
- `data/douban_cookies.pkl`

## 4. 路径兼容

应用支持从旧根目录数据自动迁移到 `data` 目录，避免升级后数据丢失。
